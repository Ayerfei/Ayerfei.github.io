将hugo部署到Github Pages
@ · Oct 13, 2019 · 1 min read · Update at Oct 13, 2019

以下以 win 10 为例

  1. 下载hugo和配置环境变量

https://github.com/gohugoio/hugo/releases

从以上网址找出属于自己的电脑版本的zip解压

解压后的目录如下：

新建bin目录，将hugo.exe移到bin目录下

配置环境变量，打开“此电脑 -> 右键点击‘属性’ -> 高级系统设置 -> 环境变量 -> 编辑系统变量Path”

将hugo.exe 所在目录添加进Path，如我的电脑则添加“D:\Hugo\bin”

  1. 下载git和配置环境变量

下载git参考：https://blog.csdn.net/qq_32786873/article/details/80570783

配置环境变量参考：https://blog.csdn.net/EXLsunshine/article/details/18939329

  1. 配置个人博客

进入hugo所在文件夹，空白处右键点击Git Bash Here

（1）新建网址：

hugo new site 网址名

（2）添加博客主题，这里以ananke为例：

cd 网址名

git init

git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke

echo 'theme = "ananke"' >> config.toml

（3）添加博客内容

hugo new posts/my-first-post.md

这一步要注意此时生成的文件是作为草稿的，不会出现子啊配置成功的Github Pages，需要打开编辑为”draft: false”

（4）启动服务器

hugo server -D

在浏览器打开：http://localhost:1313/

  1. 部署到Github Pages

在github上新建repository，命名为 .github.io

进入hugo所在文件夹Git Bash

hugo
cd public
git init
git add .
git commit -m "first commit"
git remote add origin git@github.com:git@github.com:username/username.github.io.git
git push -u origin master

打开个人博客主目录下的config.toml，编辑baseUrl为“https://username.github.io”

部署成功

因为每次更新一次都要输入git add / commit /push等等操作，所以将这些操作保存到个人博客主目录下的新建的deploy.sh

复制以下内容到deploy.sh：

#!/bin/bash

echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"
msg="rebuilding site `date`"
if [ $# -eq 1 ]
then msg="$1"
fi

hugo -t=we -D

cd public
git add -A
git commit -m "$msg"
git push --force -u origin master

cd ../

给deploy.sh添加可执行权限

在个人博客主目录下Git Bash：

chmod +x deploy.sh

之后每次新建文章

hugo new posts/hugo-start.md

将hugo-start.md中的“draft： true”改为“draft： false”，继续：

hugo
./deploy.sh
